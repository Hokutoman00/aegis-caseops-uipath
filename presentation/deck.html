The problem · Business impact

Security automation fails in two ways: too slow, or too trigger-happy.

  • SOC teams drown in alerts. But the scary failure of automation isn't speed — it's an AI isolating a production server on one flaky signal.
  • The question we built around: what would it take for a CISO to actually trust an automated incident pipeline?
  • Our answer: (1) make the verdict deterministic · (2) keep every evidence gap visible · (3) make it re-auditable by anyone with the raw alert.

Innovation · Completeness

Four declared invariants — each with a negative test

1
No remediation without approval — the approval-required edge is structurally the only path into remediation on the BPMN graph.
2
Gaps propagate, never swallowed — a failed enrichment tool appears as the same string at intake, approval, and postmortem.
3
Only the deterministic core writes the verdict — the LLM is read-only, enforced at three layers (prompt/tool, BPMN binding, blind grader).
4
One input schema — every scenario shares the same alert contract.
e.g. the low case must NOT require approval — asserted04 / 12

When tools degrade, the machine gets more careful — not more confident

Proof of trust · Re-auditability

A 60-line blind grader recomputes the verdict from the raw alert

  • verify_verdict.py takes the raw alert + triage output and diffs the 5 verdict fields.
  • Whatever changes a verdict — an LLM, a workflow edit, a hand patch — it fails loudly with a non-zero exit.
  • Demonstrated: intact → "verdict intact" exit 0 · tampered severity + approval bypass → "VERDICT TAMPERED (2 fields)" exit 1.
  • Judges can try it themselves — tamper with any verdict field and watch it get caught.

Platform usage

The UiPath SDK as the verdict substrate — not demo decoration

  • Three coded agents (triage / remediation / postmortem) = plain uipath SDK, zero LLM in the decision path, dataclass-in/out.
  • uipath eval made real: a custom evaluator + eval set reproduce the verdict contract — all 3 demo scenarios score 1.0, a tampered expected-set drops one to 0.5.
  • Maestro BPMN (XOR gateway + 4 condition edges) · Orchestrator queue · Agent Builder + Context Grounding.
  • CLI: uipath init → run → eval → register → (auth → deploy → invoke). Local eval holds even before cloud gates.
Aegis CaseOps09 / 12
Technical depth

Averaging let one 0.9 signal look authoritative. We killed that.

  • Corroboration cap: confidence = min(avg, 0.40 + 0.15 × signal_count).
  • More independent signals monotonically raise the ceiling. A single signal caps at 0.55 → cannot auto-escalate.
  • Severity thresholds 5/9/14 are justified by synthetic boundary tests (4/5, 8/9, 13/14) — not overfit to the demo fixtures.
  • 38 pytest cases pin the 3 demos + 4 invariants (with negative asserts) + threshold boundaries + tamper detection.
